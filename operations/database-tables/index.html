    Database Tables¶

    The broker uses MariaDB for three purposes: WAL fallback, recovery state tracking, and ledger lifecycle management. All tables are created dynamically at runtime.

    Table Inventory¶

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA
skinparam defaultFontName Inter

package "MariaDB: appdb" {

  package "Per-Queue Tables (dynamic)" {
    rectangle "enqueue_{queue}_tbl\n(one per queue)" as et
    rectangle "dequeue_{queue}_tbl\n(one per queue)" as dt
  }

  package "Shared Tables" {
    rectangle "broker_ledger_tracking\n(single table)" as blt
  }

  et ..> dt : message_id reconciliation\n(SET DIFFERENCE = queue state)
  blt --> et : drives recovery scan
  blt --> dt : drives recovery scan
}
@enduml

    1. enqueue_{queue}_tbl¶

    One table per queue. Stores every enqueue (publish) event extracted from BookKeeper ledgers. This table is the source of truth for "what was put into the queue."

    DDL¶

    CREATE TABLE IF NOT EXISTS enqueue_{queue}_tbl (
    message_id     VARCHAR(40)  NOT NULL,
    timestamp      BIGINT       NOT NULL,
    payload        BLOB,                   -- full message body
    queue_instance VARCHAR(50)  NOT NULL,
    queue_partition VARCHAR(50) NOT NULL,
    ledger_id      BIGINT       NOT NULL,
    last_modified  TIMESTAMP    NOT NULL
                   DEFAULT CURRENT_TIMESTAMP
                   ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (message_id, queue_partition),
    INDEX idx_queue_instance (queue_instance)
) ENGINE=InnoDB
  ROW_FORMAT=DYNAMIC
  PARTITION BY LIST COLUMNS(queue_partition) (
      PARTITION p_default VALUES IN ('default')
  );

    Columns¶

    Column Type Description
    message_id VARCHAR(40) UUID — unique per message; part of composite PK
    timestamp BIGINT Message creation time (epoch milliseconds)
    payload BLOB Full serialised message body (BigQueueTextMessage protobuf)
    queue_instance VARCHAR(50) The broker's file UUID for this queue instance; indexed
    queue_partition VARCHAR(50) Partition identifier; part of composite PK and partition key
    ledger_id BIGINT BookKeeper ledger ID this message came from
    last_modified TIMESTAMP Auto-updated on any row change

    Key Operations¶

    Operation When SQL Pattern
    INSERT WAL fallback (BookKeeper failure) or ledger scan during recovery INSERT INTO enqueue_{q}_tbl (...) VALUES (...)
    INSERT IGNORE Retry path — avoid duplicate key violation INSERT IGNORE INTO ...
    Batch INSERT Ledger processing — bulk load entries batchInsert() via JDBC batch
    SELECT FOR UPDATE SKIP LOCKED Dequeue: claim messages without contention SELECT ... FOR UPDATE SKIP LOCKED LIMIT N
    DELETE … RETURNING Atomic dequeue: remove row and return payload in one statement DELETE FROM enqueue_{q}_tbl WHERE message_id IN (...) RETURNING message_id, payload, ...
    DELETE JOIN Scrub: remove rows that already have a matching dequeue record DELETE e FROM enqueue_{q}_tbl e JOIN dequeue_{q}_tbl d ON e.message_id = d.message_id
    SELECT COUNT Recovery progress counting SELECT COUNT(*) WHERE queue_instance = ?

    Role in Message Lifecycle¶

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA

title "enqueue_tbl Role in Message Lifecycle"

start

:BookKeeper write (WAL, PUBLISH action);

if (BookKeeper succeeds?) then (yes)
  :Ledger scan later migrates\nentry to enqueue_tbl\n(cheap storage);
else (no — SQL fallback)
  :Direct INSERT into enqueue_tbl\n(immediate SQL fallback);
endif

:Row persists until message is consumed;

:DELETE … RETURNING\n(atomic dequeue);

:Row removed — message delivered;

stop
@enduml

    2. dequeue_{queue}_tbl¶

    One table per queue. Records every dequeue (consume) event. No payload column — only the message identifier is needed to track that a message was consumed.

    DDL¶

    CREATE TABLE IF NOT EXISTS dequeue_{queue}_tbl (
    message_id      VARCHAR(40)  NOT NULL,
    timestamp       BIGINT       NOT NULL,
    queue_instance  VARCHAR(50)  NOT NULL,
    queue_partition VARCHAR(50)  NOT NULL,
    ledger_id       BIGINT       NOT NULL,
    last_modified   TIMESTAMP    NOT NULL
                    DEFAULT CURRENT_TIMESTAMP
                    ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (message_id, queue_partition),
    INDEX idx_queue_instance (queue_instance)
) ENGINE=InnoDB
  ROW_FORMAT=DYNAMIC
  PARTITION BY LIST COLUMNS(queue_partition) (
      PARTITION p_default VALUES IN ('default')
  );

    Columns¶

    Column Type Description
    message_id VARCHAR(40) UUID — matches enqueue_tbl.message_id; part of composite PK
    timestamp BIGINT Dequeue time (epoch milliseconds)
    queue_instance VARCHAR(50) Queue instance file UUID; indexed
    queue_partition VARCHAR(50) Partition identifier; part of composite PK and partition key
    ledger_id BIGINT BookKeeper ledger ID of the dequeue WAL entry
    last_modified TIMESTAMP Auto-updated on any row change

    No payload column

    The dequeue_tbl intentionally has no payload column. Storing the message body here would double the storage footprint. The payload is only kept in enqueue_tbl until delivery is confirmed.

    Key Operations¶

    Operation When SQL Pattern
    INSERT IGNORE WAL fallback (BookKeeper failure) or ledger scan — record that message was consumed INSERT IGNORE INTO dequeue_{q}_tbl (...) VALUES (...)
    Batch INSERT Ledger processing — bulk dequeue event migration batchInsert()
    DELETE by instance Queue instance cleanup or pod restart DELETE FROM dequeue_{q}_tbl WHERE queue_instance = ?
    DELETE JOIN Scrub: remove dequeue records that no longer have matching enqueue records DELETE d FROM dequeue_{q}_tbl d JOIN enqueue_{q}_tbl e ON d.message_id = e.message_id
    SELECT 1 LIMIT 1 Existence check for a partition hasAnyDataInPartition()

    3. broker_ledger_tracking¶

    Single shared table across all queues. Tracks every BookKeeper ledger that was created, whether it has been fully processed (entries migrated to enqueue_tbl / dequeue_tbl), and processing statistics.

    DDL¶

    CREATE TABLE IF NOT EXISTS broker_ledger_tracking (
    queue                   VARCHAR(50),
    queue_partition         VARCHAR(50),
    file                    CHAR(36),      -- UUID
    ledger_id               BIGINT,
    processed               BOOLEAN,
    message                 TEXT,
    total_record_processed  BIGINT,
    total_enqueue_record    BIGINT,
    total_dequeue_record    BIGINT,
    time_elapsed            BIGINT,        -- milliseconds
    PRIMARY KEY (queue_partition, file, ledger_id)
);

    Columns¶

    Column Type Description
    queue VARCHAR(50) Queue name (used for filtering)
    queue_partition VARCHAR(50) Partition; part of composite PK
    file CHAR(36) Broker instance file UUID; part of composite PK
    ledger_id BIGINT BookKeeper ledger ID; part of composite PK
    processed BOOLEAN false = entries not yet migrated to SQL tables; true = done
    message TEXT Status message or error description
    total_record_processed BIGINT Count of ledger entries successfully processed
    total_enqueue_record BIGINT Count of PUBLISH entries found in this ledger
    total_dequeue_record BIGINT Count of CONSUME entries found in this ledger
    time_elapsed BIGINT Wall-clock time to process this ledger (ms)

    Key Operations¶

    Operation When SQL Pattern
    INSERT New ledger created in BookKeeper INSERT INTO broker_ledger_tracking (...) VALUES (...) with processed=false
    SELECT Lookup ledger state SELECT * ... WHERE queue_partition=? AND file=? AND ledger_id=?
    SELECT FOR UPDATE SKIP LOCKED Recovery: claim a ledger for processing (prevents duplicate processing on multi-pod) SELECT ... FOR UPDATE SKIP LOCKED
    SELECT COUNT Count pending (unprocessed) ledgers for a queue SELECT COUNT(*) WHERE queue=? AND processed=0
    SELECT with NOT IN Find ledgers belonging to old/stale broker instances WHERE file NOT IN (...)
    UPDATE Mark ledger processed + write stats UPDATE ... SET processed=true, total_record_processed=?, time_elapsed=? WHERE ...
    UPDATE conditional Only mark processed if not already done (idempotent) UPDATE ... WHERE ... AND (processed IS NULL OR processed=FALSE)
    DELETE Remove fully processed ledger tracking row DELETE WHERE queue_partition=? AND file=? AND ledger_id=?

    Role in Recovery¶

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA
skinparam sequenceMessageAlign center

title "broker_ledger_tracking in Recovery Flow"

participant "RecoveryService" as rs
database "broker_ledger_tracking" as blt
database "Apache BookKeeper" as bk
database "enqueue_{q}_tbl" as et
database "dequeue_{q}_tbl" as dt

rs -> blt : SELECT WHERE queue=? AND processed=FALSE\nFOR UPDATE SKIP LOCKED
blt --> rs : [unprocessed ledger rows]

loop for each unprocessed ledger
  rs -> bk : open ledger (ledgerId)
  rs -> bk : readEntries(0, lastAddConfirmed)

  loop for each entry
    alt WALAction.PUBLISH
      rs -> et : INSERT INTO enqueue_{q}_tbl (batch)
    else WALAction.CONSUME
      rs -> dt : INSERT IGNORE INTO dequeue_{q}_tbl (batch)
    end
  end

  rs -> blt : UPDATE SET processed=TRUE,\ntotal_enqueue_record=N,\ntotal_dequeue_record=M,\ntime_elapsed=T
end
@enduml

    Table Partitioning¶

    Both enqueue_tbl and dequeue_tbl use LIST COLUMNS partitioning on queue_partition:

    PARTITION BY LIST COLUMNS(queue_partition) (
    PARTITION p_default VALUES IN ('default')
);

    New partitions are added dynamically as new queue partitions are created:

    ALTER TABLE enqueue_{queue}_tbl
  ADD PARTITION (PARTITION p_{sanitized_id} VALUES IN ('{partition_id}'));

    This limits full-table scans to a single partition when filtering by queue_partition.

    Table Lifecycle¶

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA

title "Table Lifecycle — Queue Create to Delete"

start

:Queue created\n(QueueContainer.getQueueOrCreateIfNotExist);

fork
  :CREATE TABLE enqueue_{queue}_tbl;
fork again
  :CREATE TABLE dequeue_{queue}_tbl;
end fork

:INSERT into broker_ledger_tracking\nwhen first ledger is created;

repeat
  :Normal operation;\n:enqueue rows added on publish,\ndequeue rows added on consume;
repeat while (queue active?)

:DELETE command broadcast\n(BrokerCommander → all pods);

fork
  :DROP TABLE enqueue_{queue}_tbl;
fork again
  :DROP TABLE dequeue_{queue}_tbl;
end fork

:DELETE FROM broker_ledger_tracking\nWHERE queue=?;

stop
@enduml
