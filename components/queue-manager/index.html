    Queue Manager¶

    The queue manager layer is the core of the broker. It coordinates message ingestion, buffering, scheduling, and delivery.

    Class Hierarchy¶

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA
skinparam defaultFontName Inter

interface QueueManager {
  + publish(queue, message, tps): Mono<Void>
  + consume(queue): Flux<Message>
  + registerListener(queue, host, port, tps)
  + deregisterListener(queue, host, port)
  + size(queue): long
  + inFlight(queue): int
}

class BigQueueManager {
  - containers: Map<String, QueueContainer>
  - listeners: Map<String, ListenerHolder>
  - recordLedger: RecordLedger
  - reactiveHandler: ReactiveMessageHandler
  - rateLimiter: RedisGlobalRateLimiter
  + publishtoPipe(queue, message, tps): Mono<Void>
  + handleMessage(bytes, consumer): Mono<DeliveryAck>
  + tryCreateQueue(queue): QueueHolder
  + requestQueueDeletion(queue): Mono<Void>
}

class QueueContainer {
  - holders: ConcurrentHashMap<String, QueueHolder>
  + getQueueOrCreateIfNotExist(queue): QueueHolder
  + removeQueue(queue): void
}

class QueueHolder {
  - queue: BigQueueImpl
  - rateLimiter: RateLimiter
  - scheduler: ScheduledFuture<?>
  - inFlightCount: AtomicInteger
  - tps: int
  + enqueue(bytes): void
  + pipe(bytes): boolean
  + dequeue(): byte[]
  + size(): long
  + inFlight(): int
  + createScheduleIfNotExist(): void
  + cancelSchedule(): void
}

class ListenerHolder {
  - consumers: CopyOnWriteArrayList<NamedConsumer>
  - blacklist: Set<String>
  - rrIndex: AtomicInteger
  + addConsumer(host, port): void
  + removeConsumer(host, port): void
  + selectConsumer(): Optional<NamedConsumer>
  + blacklist(consumer): void
  + isBlacklisted(consumer): boolean
}

class NamedConsumer {
  + host: String
  + port: int
  + tps: int
  + failureCount: AtomicInteger
  + lastFailure: Instant
}

QueueManager <|.. BigQueueManager
BigQueueManager --> QueueContainer
BigQueueManager --> ListenerHolder
QueueContainer --> QueueHolder
ListenerHolder --> NamedConsumer
@enduml

    BigQueueManager — Core Logic¶

    BigQueueManager is the central orchestrator. All publish and consume operations pass through it.

    publishtoPipe() — Publish with Pass-Through¶

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA
skinparam sequenceMessageAlign center

title "publishtoPipe() — With Pass-Through Decision"

participant "BigQueueManager" as bqm
participant "QueueContainer" as qcon
participant "QueueHolder" as qh
participant "ListenerHolder" as lh
participant "RedisGlobalRateLimiter" as rrl
participant "RecordLedger" as rl
participant "ReactiveMessageHandler" as rmh

[-> bqm : publishtoPipe(queue, message, tps)

bqm -> qcon : getQueueOrCreateIfNotExist(queue)
qcon --> bqm : QueueHolder

bqm -> lh : selectConsumer()

alt No consumer
  bqm -> qh : enqueue(message)
  bqm ->> rl : writeAheadLog(PUBLISH) [async]
else Consumer available
  bqm -> rrl : tryConsume(queue, 1)
  alt Rate limit OK
    bqm -> qh : inFlight()
    alt In-flight < threshold
      bqm -> rmh : deliver(message, consumer) [pass-through]
      bqm ->> rl : writeAheadLog(PUBLISH) [async]
    else In-flight exceeded
      bqm -> qh : enqueue(message)
      bqm ->> rl : writeAheadLog(PUBLISH) [async]
    end
  else Rate limit exceeded
    bqm -> qh : enqueue(message)
    bqm ->> rl : writeAheadLog(PUBLISH) [async]
  end
end
@enduml

    QueueHolder — Per-Queue State Machine¶

    QueueHolder manages all state for a single queue on a single pod. It owns:

    • The BigQueueImpl on disk
    • The Guava RateLimiter for local TPS enforcement
    • The ScheduledFuture that periodically drains the queue
    • An AtomicInteger tracking in-flight message count
    @startuml
!theme plain
skinparam backgroundColor #FAFAFA

title "QueueHolder Internal State"

state "Idle\n(empty queue)" as idle
state "Draining\n(scheduler active)" as draining
state "Throttled\n(rate limit hit)" as throttled
state "Paused\n(no consumers)" as paused

[*] --> idle : created
idle --> draining : message enqueued\n+ consumer available
draining --> idle : queue empty
draining --> throttled : rate limit exceeded\n(Guava RateLimiter)
throttled --> draining : token refilled
draining --> paused : all consumers\nblacklisted / gone
paused --> draining : consumer re-registered
@enduml

    Drain Scheduler¶

    The scheduler is created lazily when the first message arrives (if no pass-through was possible). It polls BigQueue at a rate adapted to the queue's configured TPS:

    pollInterval = max(1, 1000 / tps)  // milliseconds

    ListenerHolder — Consumer Round-Robin¶

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA
skinparam sequenceMessageAlign center

title "Round-Robin Consumer Selection"

participant "ListenerHolder" as lh
participant "NamedConsumer[0]\nhost-a:8080" as c0
participant "NamedConsumer[1]\nhost-b:8080" as c1
participant "NamedConsumer[2]\nhost-c:8080" as c2

note over c2 : blacklisted (3 failures)

[-> lh : selectConsumer()
lh -> lh : rrIndex++ % activeConsumers.size()

alt index=0
  lh --> c0 : selected
else index=1
  lh --> c1 : selected
else index=2 (blacklisted)
  lh -> lh : skip, try next
  lh --> c0 : fallback to next valid
end
@enduml

    Blacklisting rules:

    Condition Action
    3+ consecutive delivery failures Consumer added to blacklist
    Consumer deregisters Consumer removed from active list
    Blacklist TTL expires (5 min) Consumer re-admitted

    QueueContainer — Lazy Initialization¶

    QueueContainer creates QueueHolder instances on demand. Before creating a new queue, it checks for residual state from a previous pod instance and cleans it up.

    @startuml
!theme plain
skinparam backgroundColor #FAFAFA
skinparam sequenceMessageAlign center

title "getQueueOrCreateIfNotExist()"

participant "QueueContainer" as qcon
participant "QueueInstanceTrackingRegistry\n(etcd)" as qitr
database "BigQueue Files" as bqf

[-> qcon : getQueueOrCreateIfNotExist(queue)
qcon -> qcon : check holders map

alt Already exists
  qcon --> [: return existing QueueHolder
else New queue
  qcon -> qitr : checkAndClaimInstance(queue)
  alt Residual from previous pod
    qitr --> qcon : residual detected
    qcon -> bqf : clean up stale files
    qcon -> qitr : release old instance
  end
  qcon -> bqf : initialise BigQueueImpl
  qcon -> qitr : register new instance
  qcon --> [: return new QueueHolder
end
@enduml
